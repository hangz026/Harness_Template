# Lessons 说明

`lessons/` 用于沉淀独立于具体项目的、可复用的实现经验、常见陷阱和检查清单。

## 适用范围

- 适合记录不依赖某个单一项目背景、可迁移到其他项目的工程经验
- 适合记录跨任务、跨阶段可复用的技术经验
- 适合记录某类实现中反复出现的问题和排查路径
- 适合记录“默认值不可盲信”“集成时必须显式处理”的工程经验

## 不适用范围

- 不写当前项目专属决策、界面约束、交付边界或业务口径
- 不写项目专属需求
- 不写硬约束
- 不写架构设计本身
- 不写变更历史
- 不写一次性任务流水账

如果某条内容只有脱离当前项目后仍然成立，才适合写入 `lessons/`。
如果它依赖当前项目的需求、理论文档、实现结构或交付方式，则应写入 `Spec.md`、`Rule.md`、`Architecture.md` 或 `STATUS.md`，而不是写入 `lessons/`。

## 使用方式

1. 当任务涉及特定技术主题时，先检查是否存在对应 lesson
2. 若存在相关 lesson，方案和编码必须受其约束
3. 若踩到新问题且具备复用价值，在获批后补充新的 lesson

## 与其他文档的关系

- `Rule.md`：硬约束
- `Spec.md`：需求与交付物
- `Architecture.md`：当前实现结构与演化边界
- `lessons/`：独立于项目的可复用实现经验
- `CHANGELOG.md`：变更历史
